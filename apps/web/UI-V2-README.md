# easyStory UI v2 - 对话式创作工作台

## 🎯 设计理念

这次重构的核心目标是将 easyStory 从"后台管理系统"转变为"对话式创作工作台"。

### 三个核心转变

1. **从"填表配置"到"对话生成"**
   - ❌ 用户填写 20 个表单字段
   - ✅ 用户说"我想写一个修仙小说"，AI 生成完整配置

2. **从"后台管理"到"创作伙伴"**
   - ❌ 进入配置中心 → 找到 Skills → 点击新建 → 填写 YAML
   - ✅ 在创作时说"帮我加个人物塑造的能力"，AI 自动生成 Skill

3. **从"固定流程"到"灵活编排"**
   - ❌ 必须按顺序：设定 → 大纲 → 开篇 → 章节
   - ✅ 随时可以：聊想法 → 生成片段 → 调整设定 → 继续写

## 🚀 快速开始

### 1. 启动开发服务器

```bash
cd apps/web
npm run dev
```

### 2. 访问 Demo 页面

打开浏览器访问：`http://localhost:3000/demo`

你会看到三个页面的演示：
- **书架 Lobby**：项目入口，快速开始
- **对话起稿 Incubator**：用对话生成项目配置
- **创作桌面 Studio**：三栏布局的写作界面

## 📄 页面详解

### 1. Lobby（书架）

**设计理念**：书架是创作入口，不是项目管理后台

**核心功能**：
- 快速开始区：一键进入对话起稿
- 项目卡片：突出作品属性（题材、字数、更新时间）
- 助手区：直接展示能力，不是配置列表

**布局**：
```
┌──────────────────────────────────────────┐
│  侧栏                主内容区              │
│  ├─ 快速开始         ├─ 页面标题          │
│  │  ├─ 对话起稿      ├─ 搜索框            │
│  │  ├─ 模板起稿      ├─ 统计卡片          │
│  │  └─ 导入作品      └─ 项目卡片网格      │
│  ├─ 导航                                  │
│  │  ├─ 我的作品                           │
│  │  ├─ 我的助手                           │
│  │  ├─ 模板库                             │
│  │  └─ 回收站                             │
│  └─ 提示信息                              │
└──────────────────────────────────────────┘
```

### 2. Incubator（对话起稿）

**设计理念**：对话式交互，不是填表

**核心功能**：
- 对话区：用自然语言描述想法
- AI 生成：边聊边生成配置文档
- 文档预览：右侧实时显示生成的文档
- 灵活调整：可以继续对话调整，也可以直接编辑文档

**布局**：
```
┌──────────────────────────────────────────┐
│  对话区                文档预览区          │
│  ├─ 顶部导航           ├─ 标题            │
│  ├─ 消息列表           ├─ 文档列表        │
│  │  ├─ AI 消息         │  ├─ 项目设定     │
│  │  ├─ 用户消息        │  ├─ 创作规则     │
│  │  └─ 生成的文档      │  └─ 人物设定     │
│  └─ 输入框             └─ 操作按钮        │
└──────────────────────────────────────────┘
```

### 3. Studio（创作桌面）

**设计理念**：三栏布局，文档树 + 编辑器 + AI 助手

**核心功能**：
- 左栏：文档树（设定/规则/人物/大纲/章节）
- 中栏：Markdown 编辑器（写作主舞台）
- 右栏：AI 助手（对话/续写/改写/工具）

**布局**：
```
┌──────────────────────────────────────────┐
│  文档树      编辑器          AI 助手      │
│  ├─ 设定     ├─ 工具栏      ├─ 快捷工具  │
│  │  ├─ 项目  ├─ 编辑区      │  ├─ 续写   │
│  │  ├─ 规则  │  (Markdown)  │  ├─ 改写   │
│  │  ├─ 人物  └─ 状态栏      │  └─ 润色   │
│  │  └─ 世界观                ├─ 上下文   │
│  ├─ 大纲                     ├─ Skills   │
│  └─ 正文                     └─ 对话输入 │
└──────────────────────────────────────────┘
```

## 🎨 视觉设计

### 色彩系统

```css
/* 温暖基调 */
--color-canvas: #faf9f6;      /* 画布背景 */
--color-paper: #ffffff;        /* 纸面白色 */
--color-sidebar: #f5f4f0;      /* 侧栏米色 */

/* 强调色 */
--color-accent: #4a7c59;       /* 主色调（绿色） */
--color-accent-hover: #3a6c49; /* 悬停色 */
--color-accent-light: #e8f3ed; /* 浅色背景 */
```

### 字体系统

```css
--font-serif: "Noto Serif SC", Georgia, serif;  /* 标题字体 */
--font-sans: -apple-system, sans-serif;          /* 正文字体 */
--font-mono: "SF Mono", "Fira Code", monospace;  /* 代码字体 */
```

### 组件样式

- **按钮**：圆角 8px，悬停时轻微上移
- **卡片**：圆角 12px，悬停时显示阴影
- **输入框**：圆角 8px，聚焦时显示浅色边框
- **对话气泡**：圆角 8px，带淡色背景

## 💡 设计原则

1. **对话优先**：用自然语言描述需求，AI 生成配置
2. **文档化配置**：所有配置都是 Markdown 文档，易读易改
3. **渐进式披露**：初级用户只看对话，高级用户可以深入
4. **创作优先**：界面像写作工具，不像后台管理
5. **灵活编排**：不强制流程，随时可以调整
