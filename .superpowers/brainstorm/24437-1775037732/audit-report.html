easyStory 前端 UI 全面审查报告

基于源码逐页分析，按严重程度排序

🔴 严重问题（视觉/布局明显异常）

1

项目设置页：侧边栏双重 sticky + 双重圆角

project-settings-page.tsx:92 外层已经 sticky top-6，而 project-settings-sidebar.tsx:43 内层又 sticky top-6，造成双重定位冲突。
同时侧边栏 rounded-[24px] 在设置页语境下过于膨胀（与 lobby 的 rounded-[28px] 不是同一套圆角系统）。

2

GuardedLink 按钮：巨量重复内联样式

project-settings-sidebar.tsx:119-143 三个 GuardedLink 各自包含一模一样的 ~200 字符内联 className。
这不仅是维护灾难，任何微调都需要改三处。应提取为全局类（如 ink-link-button）或 Tailwind @apply。

3

Studio 页面：硬编码三栏 + body 纹理重复

studio-page.tsx:118 用 [grid-template-columns:260px_1fr_380px] 无响应式断点，平板和窄屏直接裂开。
studio-page.tsx:119 手动重复了 body 的噪声纹理 SVG（已在 globals.css body::before 定义），造成双层纹理叠加。

4

Lobby 侧边栏：搜索框自定义样式 vs ink-input 不一致

lobby-page.tsx:113 搜索框使用了一套独立的圆角 (rounded-[18px])、高度 (min-h-12) 和内边距，与全局 .ink-input 的 rounded-md min-h-9 完全不同。
造成同一产品内两套输入框视觉系统。

🟡 重要问题（不一致 / 可维护性差）

5

圆角系统混乱：至少 5 套并行体系

globals.css 定义了 --radius-xs/sm/md/lg (3-8px)，但各页面硬编码了 rounded-[18px], rounded-[22px], rounded-[24px], rounded-[26px], rounded-[28px]。
这些数值缺少对应的 token 或 Tailwind 配置，几乎不可能统一调整。

6

阴影系统不统一

Tailwind 配置了 shadow-xs/sm/md/lg，但各组件大量使用 shadow-[0_18px_48px_rgba(...)] 等硬编码值。
特别是 auth、engine、lab、incubator 共用了一套"hero 卡片阴影"，但每个文件都内联重复。

7

Engine 主按钮：独立于 ink-button 体系

engine-page-shell.tsx:103 和 auth-form.tsx:200 的主按钮都用了 min-h-[50px] bg-gradient-to-br from-[var(--accent-primary)] to-[#47665a]，
与 .ink-button 的 32px 高度 + 纯色背景完全不同。同一产品两套主按钮。

8

标签文案样式 "accent-secondary + 11px tracking" 被复制了 12+ 次

几乎每个 Hero 区域都有 text-[var(--accent-secondary)] text-[11px] tracking-[0.28em] uppercase 这个 pattern。
应该提取为 .label-overline 或类似的全局类。

🟢 次要问题（打磨细节）

9

Lobby 项目卡片颜色值硬编码

lobby-project-shelf.tsx:97 的阴影 rgba(61,61,61,0.07) 和背景 rgba(255,253,251,0.92) 没有走 CSS 变量。
如果以后要做暗色模式，这些全是改不动的遗留。

10

WorkspaceShell 响应式不足

workspace-shell.tsx:74 在 studio 模式下是 w-full，非 studio 模式下是 w-[min(100%-2.5rem,1560px)]。
但 Header 内部的三栏 grid 在窄屏下会挤压导航项，没有折叠或 overflow 处理。

11

tailwind.config.ts 缺少大圆角 token

目前只有 xs(3px) sm(4px) md(6px) lg(8px)。
需要补充 xl(12px) 2xl(16px) 3xl(20px) 4xl(24px) 5xl(28px) 来收编各页面硬编码的大圆角值。

12

Lobby/Engine/Lab 共享"Hero Card"模式未提取

Auth, Engine, Lab, Incubator 都有相同的 Hero 卡片结构（圆角 28px + 半透明白底 + backdrop-blur + 大阴影）。
应该提取为 .hero-card 全局类或共享组件。

📋 建议实施顺序

  1. P0 — 修复 Studio 响应式 + 移除重复纹理（#3）
  2. P0 — 修复项目设置侧边栏双重 sticky（#1）
  3. P1 — 补充 Tailwind 大圆角 token + 收编硬编码值（#5, #11）
  4. P1 — 提取 hero-card / label-overline / ink-link-button 全局类（#2, #6, #8, #12）
  5. P1 — 统一按钮体系：把 Engine/Auth 大按钮纳入 ink-button 变体（#7）
  6. P2 — Lobby 搜索框统一 + 项目卡片颜色走变量（#4, #9）
  7. P2 — WorkspaceShell Header 响应式折叠（#10）
