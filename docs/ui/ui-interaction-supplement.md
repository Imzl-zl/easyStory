## 一、整体诊断

当前 UI 白皮书定义了 **4 个核心页面 + 2 个子视图**：

| 页面 | 覆盖功能 |
|------|---------|
| 灵感长廊 (Lobby) | 书架管理、开启新卷 |
| 创作孵化器 (Incubator) | 对话式设定 |
| 创作砚台 (Studio) | 编辑器、AI 对话 |
| 逻辑织网 (Engine) | 工作流编排、节点配置 |
| 文风实验室 (Lab) | 小说分析、风格提取 |
| 审核精修 (Review & Diff) | 对比视图 |

但功能需求（原设计 + 补充方案）定义了 **至少 15 个功能区域**，其中 **9 个没有对应的 UI 设计**。

## 四、技术文档不一致

| 不一致点 | 文档 A | 文档 B | 建议 |
|---------|--------|--------|------|
| 实时通信 | tech-stack: "SSE 优先，WebSocket 延后" | ui-design + 原设计: WebSocket | 统一为 **SSE 优先**（MVP 足够，降低复杂度）|
| 数据库表名 | database-design: `WorkflowRun` / `NodeRun` | 实施计划: `WorkflowExecution` / `NodeExecution` | 统一为一套命名 |
| 审核输出 | config-format: Agent 的 `output_schema` 自由定义 | 补充方案 #23.1: 强制 `ReviewResult` 结构 | 统一为 **ReviewResult 强制 Schema** |
| 导出存储 | database-design: "导出文件存文件系统" | architecture: "v0.1 存数据库 BLOB" | 统一为 **文件系统**（database-design 更新） |
| 内容表结构 | database-design: `Content` 有 `content` 字段 | 实施计划: `Content` + `ContentVersion` 分离 | 统一为 **ContentVersion 分离**（已在补充方案确认） |

## 五、交互流程总览（补充后）

```
注册/登录
  ↓
灵感长廊 (Lobby)
  ├── 开启新卷 → 创作孵化器 (Incubator) → 对话/模板/手填 → 创建项目
  ├── 继续执笔 → 创作砚台 (Studio)
  │     ├── 章节导航（左侧）
  │     ├── Markdown 编辑器（中间）
  │     ├── AI 对话助手（右侧/侧滑）
  │     ├── 上下文胶囊（顶部）→ 事实库查看
  │     └── 版本历史（侧面板）
  ├── 工作流 → 逻辑织网 (Engine)
  │     ├── 编排模式（静态配置）
  │     ├── 校验 → 错误列表展示
  │     ├── Dry-run → 成本预估确认
  │     ├── 执行模式（实时监控 + 进度 + Token 消耗）
  │     ├── 章节任务列表（大纲拆分后确认）
  │     └── 节点详情（上下文报告 + Prompt 回放 + 日志）
  ├── 文风分析 → 文风实验室 (Lab)
  │     ├── 上传 → 采样 → 分析
  │     ├── 分析报告（可编辑）
  │     ├── Skill 试写 → 保存
  │     └── Skill 作用域选择（全局/项目）
  ├── 导出 → 导出对话框（格式/范围/版本/排版）
  ├── 回收站 → 已删除项目（恢复/永久删除）
  └── 设置 → 账户/模型凭证/预算/系统
```
