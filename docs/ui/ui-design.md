## 1. 文档定位

本文档定义 easyStory 当前生效的产品心智、页面语义和布局原则。

真值边界如下：

- 产品语义与布局原则：`docs/ui/ui-design.md`
- 前端实施约束：`docs/ui/implementation-guide.md`
- 运行时样式 token 真值：`apps/web/src/app/globals.css`
- 实际路由入口：`apps/web/src/app/**`
- 页面与模块实现：`apps/web/src/features/**`
- 共享 UI 原件：`apps/web/src/components/ui/**`
- `docs/ui/ui-design-v2.md` 只保留为探索草图，不是运行时真值

核心转变：

- 从“填表配置”到“对话生成”
- 从“后台管理”到“创作伙伴”
- 从“固定流程”到“灵活编排”

## 2. 产品心智

### 2.1 核心理念

```
┌─────────────────────────────────────────────────────────┐
│  easyStory = 创作工作台 + AI 协作伙伴 + 文档化配置      │
│                                                         │
│  用户不应该：                                           │
│  ❌ 填写大量表单                                        │
│  ❌ 手动理解系统资源树                                  │
│  ❌ 在后台管理视角里找创作入口                          │
│                                                         │
│  用户应该：                                             │
│  ✅ 通过对话整理故事想法                                │
│  ✅ 在文档与正文里直接修改创作内容                      │
│  ✅ 在创作流程中自然使用 AI 能力                        │
└─────────────────────────────────────────────────────────┘
```

### 2.2 三层交互范式

| 层级 | 交互方式 | 适用场景 |
|---|---|---|
| 对话层 | 自然语言对话 | 起稿、追问、续写、修改 |
| 文档层 | Markdown / 结构化正文 | 设定、规则、章节、版本 |
| 可视层 | 页面与状态界面 | 项目入口、任务推进、洞察结果 |

### 2.3 文案口径

- 默认说“作品、起稿、创作、推进、洞察”
- 避免在主路径里暴露“资源、控制面、配置中心、后台”
- 模型、工具、上下文属于创作辅助，不应抢占正文主舞台

常见对照：

| 场景 | 不推荐 | 推荐 |
|---|---|---|
| 项目列表 | 资源管理 | 我的作品 |
| 新建项目 | 创建资源 | 开始新故事 / 起稿 |
| 配置入口 | 系统配置 | 项目设置 / 我的助手 |
| 执行动作 | 任务队列 | 推进 / 批量生成 |
| 结果查看 | 数据分析 | 洞察 / 灵感库 |
| 模型选择 | 模型配置 | 对话模型 / 当前助手 |

## 5. 页面语义

### 5.1 Workspace Shell

- Lobby 使用完整顶栏，强调作品入口和路径导航
- Studio / Engine / Lab / Project Settings 使用项目级工具栏
- 顶栏要随着页面心智变化，不强行所有页面同壳

### 5.2 Lobby

Lobby 不是系统首页，而是书架首页。

- 左侧是轻导航，不是后台功能树
- 右侧主舞台优先展示作品和继续创作入口
- 项目卡片表达应接近“书卡”，不是“管理卡”
- 设置和技术动作是次级动作，不占视觉主位

### 5.3 Incubator

Incubator 是把故事想法整理成可写项目的空间。

- `AI 聊天` 是一起聊故事，不是配置向导
- `模板起稿` 是补齐必要信息，不是后台建表
- 主舞台只服务当前起稿模式，不并排堆叠多余控制区
- 反馈信息使用轻提示，不做后台告警堆叠

### 5.4 Studio

Studio 必须像写作桌面，而不是三块后台面板并排。

| 区域 | 宽度策略 | 角色 |
|---|---|---|
| 左栏 | 220-300px | 创作结构：设定 / 大纲 / 正文 |
| 中栏 | 弹性主舞台 | 正文、Markdown、预览 |
| 右栏 | 304-404px | 共创助手与上下文协作 |

核心原则：

- 正文永远是主舞台
- 三栏靠背景色、留白和密度自然分层
- 不使用强 1px 分隔线切出后台感
- 助手区是共创侧板，不是系统控制台
- 模型、工具、上传等会话配置靠近输入区，不占顶部主位

### 5.5 Engine

Engine 是作品推进室，不是 workflow 控制台首页。

- 用户先看到“推进到哪里、是否卡住、下一步做什么”
- 执行详情可以复杂，但页面入口必须是创作语义
- 技术概念只在需要时下钻，不作为默认主语言

### 5.6 Lab

Lab 是围绕作品的研究桌，不是分析记录管理页。

- 顶部先解释洞察价值，再展示筛选和焦点
- 结果以“对作品有什么帮助”的语义组织
- 技术字段必须翻译为用户语义

## 6. 视觉基线

### 6.1 风格关键词

- 温暖
- 克制
- 呼吸感
- 编辑感
- 上下文感知

### 6.2 视觉约束

- 避免大圆角
- 避免重阴影
- 避免胶囊化后台控件感
- 避免把系统操作按钮和正文放在同一视觉层级
- 避免模板味 landing page 语言

### 6.3 排版约束

- 标题允许使用 serif 增强作品感
- 正文保持高可读性 sans / serif 组合
- 重要数字使用 tabular-nums
- 保持精确对齐与克制节奏，不做花哨装饰

### 6.4 响应式

- 大屏优先保持三栏
- 中屏优先收窄助手栏，而不是牺牲正文
- 小屏按 `目录 -> 正文 -> 助手` 垂直堆叠
- 不能靠隐藏正文核心信息来适配移动端
