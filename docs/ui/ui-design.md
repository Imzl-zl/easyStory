## 0. 四柱架构总览 (The 4 Pillars)

> **设计目标**：将整个应用精简为 4 个核心顶层页面，通过"多维空间切换"而非"页面跳转"来处理交互

| 页面 | 核心功能 | 交互哲学 | 关键词 |
|------|----------|----------|--------|
| **灵感长廊 (Lobby)** | 书架管理、开启新卷、全局设置 | 静谧感：像走入私人藏书馆 | 画廊、作品感 |
| **创作砚台 (Studio)** | 沉浸编辑器、对话助手、上下文胶囊 | 专注感：除了文字，一切皆为虚幻 | 呼吸、悬浮 |
| **逻辑织网 (Engine)** | 工作流编排、节点配置、YAML 编辑 | 秩序感：像在白纸上勾勒工笔线条 | 逻辑、翻转 |
| **文风实验室 (Lab)** | 小说分析、风格提取、Skill 生成 | 洞察感：透视文字背后的数据灵魂 | 洞察、墨云 |

**架构原则**：
- 每个页面都有独特的**空间性格**（Spatial Character）
- 页面切换采用**墨迹渗透**过渡，而非生硬跳转
- 功能在需要时**晕开呈现**，不需要时沉淀为背景

## 1. 设计哲学：水墨流的三个维度

### 1.1 负空间主权 (Sovereignty of Negative Space)
- **理念**：留白不是空白，而是“呼吸”。我们拒绝使用厚重的边框和背景色块，利用间距（Margin/Padding）的疏密来建立视觉秩序。
- **落地**：界面中 85% 以上的区域应保持纯净底色。模块切换通过明度微差（2% 灰阶变化）而非物理线条。
- **气流间距系统 (Air-Flow Spacing)**：
  - **明度阶梯**：编辑器背景比侧边栏暗 2%（浅色模式：侧边栏 `#FFFFFF` → 编辑器 `#FAFAFA`；深色模式：侧边栏 `#18181B` → 编辑器 `#0C0E14`）
  - **无边界分割**：利用背景色微妙的对比产生空间感，而非生硬的线条

### 1.2 物理隐喻：纸、墨、砚 (Tactile Metaphor)
- **纸 (Paper)**：背景不是冰冷的像素，而是带有微弱噪点（Noise）和纤维感的宣纸纹理。
  - **多级纸张阴影**：Canvas 层使用微弱噪点纹理；Surface 层使用 0.5px inset shadow，模拟纸张边缘由于切割产生的微弱阴影，增强层级感而不使用边框。
- **墨 (Ink)**：AI 生成的内容不应“弹出”，而应像墨水在宣纸上**晕开（Bleeding）**。
  - **墨迹扩散动画**：文字出现前，在背景层产生极浅的底色晕染（类似宣纸湿水后的半透明感），文字随后在其上"固化"。两层动画增强"AI 在为你书写"的仪式感。
- **砚 (Stone)**：功能面板（Surface）具有磨砂质感，触感坚实。

### 1.3 瞬时触达 (Instant Accessibility)
- **理念**：符合国人“快捷、聚合”的效率需求，但保持表面的极简。
- **落地**：引入 `Ctrl + K` 全局命令中心。90% 的功能（如新建、导出、切换 Skill）不占用视觉位，通过快捷指令唤醒。
- **渐进式呈现**：侧栏支持三种状态：
  - **展开状态**：完整显示图标和文字
  - **极简状态（Icon Only）**：宽度仅 48px，显示图标，边框使用 1% 灰度微差
  - **隐藏状态**：完全收起
  - **交互**：当鼠标靠近边缘 100px 时，极简状态侧栏以 spring 动画效果轻微向内滑动，提供"触手可及"的反馈
- **呼吸式侧边栏（创作专注模式）**：
  - **触发**：用户在编辑器连续输入超过 3 秒
  - **表现**：左右侧边栏收缩至仅剩 5px 的"墨迹边缘"（不消失，保留色块感）
  - **恢复**：鼠标靠近边缘或停止输入 2 秒后，以 spring 动画平滑展开
  - **价值**：80% 的创作时间用户只面对纯净的稿纸

## 2. 全局样式参数 (The Visual Code)

### 2.1 配色系统 (Color Tokens)
| Token | 浅色 (Fresh Paper) | 深色 (Midnight Ink) | 备注 |
| :--- | :--- | :--- | :--- |
| `--canvas` | `#FAFAFA` (米白) | `#0C0E14` (深邃蓝黑) | 全局底色，带 2% Noise 纹理 |
| `--surface` | `#FFFFFF` (纯白) / `#FDFCF5` (象牙白-护眼模式) | `#18181B` (深灰) | 弹出层/主编辑面板 |
| `--ink` | `#171717` (炭黑) / `#2D2D2D` (柔和黑) | `#E4E4E7` (柔和白) | 核心正文，宋体渲染 |
| `--mist` | `#737373` (烟灰) / `#747A74` (墨绿灰) | `#71717A` (淡灰) | 辅助信息/占位符 |
| `--brand` | `#0891B2` (湖蓝) | `#22D3EE` (电光青) | 核心行动点/进度条 |
| `--danger` | `#EF4444` | `#F87171` | 节点报错/冲突提醒 |

**配色优化说明**：
- **浅色模式**：引入象牙白 `#FDFCF5` 作为护眼模式选项，带有极微弱暖色调，增加"陈年墨迹"韵味；正文使用 `#2D2D2D` 替代纯黑，减少视觉疲劳
- **深色模式**：背景从纯黑 `#09090B` 调整为深邃蓝黑 `#0C0E14`，更好衬托电光青品牌色
- **墨绿灰**：辅助文字使用带有墨绿色相的灰色 `#747A74`，增加书卷气

### 2.2 字体排版 (Typography)
- **正文区 (Reading)**: `Source Han Serif SC` (思源宋体)，18px，1.85 行高，2em 首行缩进，letter-spacing: 0.02em，段间距 1em
- **交互区 (UI)**: `PingFang SC` (苹方)，14px，字重 500
- **代码/参数 (Mono)**: `JetBrains Mono`，12px

**字体优化说明**：
- **呼吸感排版**：增加 `letter-spacing: 0.02em` 和段间距 `1em`，提升长篇阅读的呼吸感
- **深色模式宋体优化**：在深色模式下，为宋体增加 `text-shadow: 0 0 1px rgba(255,255,255,0.1)`，增强字形丰满度，解决横划较细产生的锯齿感

### 2.3 预算可视化系统 (Budget Visualization)

**墨水池隐喻 (Ink Pool)**：
- **视觉表现**：项目设置中预算限额可视化为一方"墨水池"
- **状态变化**：
  - **充足**：墨水池满，墨色浓郁
  - **消耗中**：墨水池逐渐变浅，墨色变淡
  - **告警**：墨水池低于 20%，池底显现裂纹纹理
  - **耗尽**：墨水池干涸，显示"需续墨"提示
- **交互**：悬停墨水池显示具体数字（已用 / 总额）
- **价值**：视觉暗示远比数字提醒更具张力，符合水墨流美学

*空间性格：静谧感，像走入私人藏书馆。视觉重点：大幅封面 + 动态聚焦，强调作品感而非工具感。*

    - **安全细节**：API Key 等敏感信息默认显示为"墨迹模糊"，点击才"晕开"可见
  - **回收站入口**：书架底部悬浮"废纸篓"图标，点击展开已删除项目列表

### 3.4 创作砚台 (Studio) - 呼吸式中心布局
*空间性格：专注感，除了文字，一切皆为虚幻。视觉重点：悬浮层而非占位块，功能在需要时浮现。*

```text
┌──────────────────────────────────────────────────────────────────────────┐
│ [←] [📖 目录]      [ 🔗 上下文胶囊：大纲 | 前二章 ]      [Mode: 手动] [📤] │
├──────────────────────────────────────────────────────────────────────────┤
│ ░░                                                                    ░░ │
│ ░░               第五章：潜龙在渊                                     ░░ │
│ ░░                                                                    ░░ │
│ ░░       窗外大雪纷飞。                                                ░░ │
│ ░░       林风盯着手中的断剑，眼神冰冷。                                 ░░ │
│ ░░       "还没有结束..."他低声说道。                                   ░░ │
│ ░░                                                                    ░░ │
│ ░░       [ AI 灵感气泡: 需要我帮你润色这段环境描写吗？] [💡]            ░░ │
│ ░░                                                                    ░░ │
│ ░░                                                                    ░░ │
├──────────────────────────────────────────────────────────────────────────┤
│ [状态：2.4k 字] [实时同步中...]                       [ 💬 唤醒助手 Ctrl+J ] │
└──────────────────────────────────────────────────────────────────────────┘
```
- **布局结构：画卷索引 (Scroll Index)**
  - **左侧边栏**：不是树状图，而是一叠"堆叠的信封"，每个信封代表一章
  - **信封状态**：不同颜色/纹理表示章节状态（已审/待修/草稿）
  - **悬停交互**：浮现该章字数、状态、Token 消耗的浮层
  - **点击交互**：平滑滚动到对应内容

- **微操**：
  - **Zen Mode**：输入文字时，顶部和底部工具栏以 0.05 透明度淡出，边距处的 `░░` 代表极淡的墨迹阴影
  - **折扇侧滑层**：点击顶部的"大纲"胶囊，右侧会像"折扇"一样拉出一个半透明的侧滑层预览内容
  - **上下文胶囊**：在当前行下方展开预览，阅读完后一键收起，保持视觉连续性
  - **语境感知工具栏**：编辑器不常驻工具栏，仅当用户选中文字时，在选区上方 10px 处以 fade-up 效果弹出极简半透明模糊（Glassmorphism）工具栏
  - **弹性侧栏**：支持展开/极简/隐藏三种状态，极简状态宽度 48px，鼠标靠近时 spring 动画滑入
  - **"随笔"指令条 (In-Line Command)**：
    - **触发**：在编辑器任何位置输入 `/` 或选中一段文字
    - **表现**：直接在光标处弹出"墨迹气泡"，AI 对话、Skill 调用、设定查询全部在气泡内完成
  - **墨迹气泡 (Ink Pop-over)**：
    - **触发**：选中文字后
    - **表现**：光标下方直接浮现三个极简图标：改(Refine)、续(Continue)、谈(Chat)
    - **交互**：点击后气泡向外扩散出输入框，不跳窗，不遮挡上下文
  - **透光叠层 (Light-Table Diff)**：
    - **场景**：版本历史对比
    - **表现**：旧版本像一层半透明的薄纸覆盖在当前内容上
    - **标记**：变动部分以"红色墨迹"标出
    - **价值**：比左右对比更符合"纸笔"逻辑，认知负担更低
  - **项目设置面板**：
    - **触发**：点击顶部 ⚙️ 图标
    - **表现**：下拉面板展示预算墨水池、模型选择、项目级配置

### 3.5 逻辑织网 (Engine) - 秩序感布局
*空间性格：秩序感，像在白纸上勾勒工笔线条。视觉重点：逻辑可视化，支持表单与 YAML 源码的 3D 翻转切换。*

**布局结构：非对称黄金比例**
- **左侧索引带**：约 12% 宽度，极简文字节点树
- **中间画布**：约 50-60% 宽度，逻辑可视化区域
- **右侧配置区**：约 20-35% 动态宽度，节点参数配置

```text
┌──────────────────────────────────────────────────────────────────────────┐
│ [←] 编排模式：[ 长生路 ]                                   [模拟] [保存] │
├────────────────────────────┬─────────────────────────────────────────────┤
│  逻辑路径 (Logic Path)      │  节点详情 (Node Surface)        [表单] [YAML] │
├────────────────────────────┼─────────────────────────────────────────────┤
│  (S) 灵感提取               │  ┌──────────────────────────────────────┐  │
│      ↓                     │  │ 节点：章节生成 (node_02)              │  │
│  (G) 章节生成 [●]──────────┤  │ 关联 Skill: [ 仙侠风 v3 ]  [⚙️]         │  │
│      ↓                     │  │ ------------------------------------ │  │
│  (R) 逻辑审核               │  │ [x] 注入大纲   [x] 注入前 3 章        │  │
│      ↓                     │  │ [ ] 自动执行   [x] 需要人工确认        │  │
│  (E) 结果输出               │  └──────────────────────────────────────┘  │
├────────────────────────────┴─────────────────────────────────────────────┤
│ [ + 新增节点 ]  [ 节点库：生成 / 审核 / 转换 ]                             │
└──────────────────────────────────────────────────────────────────────────┘
```

**3D 翻转卡片交互**：
- **正面**：节点可视化卡片，显示节点名称、类型、状态
- **背面**：YAML 源码或详细配置表单
- **翻转触发**：点击右上角 `[YAML]` 按钮，节点详情卡片执行 180 度 y 轴翻转动画
- **动画**：3D flip 动画，时长 0.4s，cubic-bezier(0.4, 0, 0.2, 1)
- **价值**：节省屏幕空间，"正面交互、背面源码"的设计避免模式切换

**实时监控 (Real-time Flow)**：
- **墨滴滴落 (Dripping Progress)**：工作流运行期间，当前执行节点图标下方产生缓慢滴落的墨滴动画
- **监控小窗 (Console In-place)**：点击运行中的节点，直接在节点下方展开实时滚动日志，而非跳转新视图
- **SSE 流式增长**：进度条像水流一样平滑滑动，与 SSE 流式特性匹配

## 4. 主题与交互微操 (Theme & Physics)

### 4.1 深度主题逻辑
- **白色模式 (Fresh Paper)**：不使用纯白底色，加入 1% 的灰和噪点，模拟宣纸。强调色使用深湖蓝。
  - **护眼变体**：提供象牙白 `#FDFCF5` 选项，带有微弱暖色调
- **黑色模式 (Midnight Ink)**：背景使用深邃蓝黑 `#0C0E14`，正文字体使用柔和的 `#E4E4E7`（防眩光），并添加微弱 text-shadow 增强字形丰满度。强调色为电光青。
- **主题切换**：采用“墨迹渗透”过渡，从点击点开始，新主题色向四周圆形扩散覆盖，时长 0.6s。

### 4.2 交互微操规范
- **按钮反馈**：所有按钮悬停时不仅变色，同时配合 `scale(0.98)` 微小压感，模拟物理按键触感
- **过渡曲线**：所有动画使用 `cubic-bezier(0.4, 0, 0.2, 1)` 确保物理感一致
- **去边框化**：模块间尽量通过背景色微差（1% difference）或间距区分，必须使用线条时，颜色应与背景极其接近（白色模式下用 `#F0F0F0`）

### 4.3 针对国人习惯的“文学排版"
- **自动缩进**：开启该模式后，输入回车，下一行自动空出两个全角字符位
- **排版灯**：编辑器右下角有一个极简的圆点，绿色代表“排版规范已开启”，红色代表“存在标点误用”
- **呼吸感排版**：正文区使用 `letter-spacing: 0.02em` 和段间距 `1em`，提升长篇阅读的舒适度

## 5. 艺术化表达：技术功能的水墨化 (Artistic Visualization)

> **设计目标**：将生硬的工业风界面元素转化为符合"水墨流"美学的艺术表达

### 5.1 有机连线 (Organic Paths)

**工作流编排器中的节点连线**：
- **避免**：90 度折线、直角连接线
- **采用**：自然贝塞尔曲线，模拟水流的柔和动态
- **数据流动画**：连线上的数据流动表现为"墨滴"顺着虚线滑过，墨滴大小与数据量成正比
- **实现**：使用 SVG 路径 + CSS stroke-dasharray 动画

```
节点 A ~~~💧~~~> 节点 B
       ↑
   墨滴沿曲线滑动
```

### 5.2 手绘感图表 (Hand-drawn Data)

**文风分析的雷达图和词云**：
- **避免**：标准 Echarts 默认样式、机械化的完美线条
- **采用**：
  - **墨线绘制**：使用粗细不均的 SVG 路径，模拟毛笔笔触
  - **位移扰动 (Jitter)**：给线条增加轻微的随机位移，使其看起来像毛笔随手画在草稿纸上
  - **飞白效果**：线条边缘带有细微的纹理，模拟宣纸吸墨的不均匀性
- **配色**：使用 `--ink` 色系的渐变，从浓到淡自然过渡

**实现要点**：
```css
/* 手绘风格线条 */
.ink-line {
  stroke: var(--ink);
  stroke-width: 2px;
  stroke-linecap: round;
  filter: url(#rough-paper); /* 纸质纹理滤镜 */
}
```

### 5.3 核心原则

> **"2D 质感 + 深度物理动画"**

所有视觉元素遵循"墨水在纸上运动"的逻辑：
- **有惯性**：动画启动和停止都有缓冲，不突兀
- **有扩散**：状态变化像墨滴在宣纸上晕开
- **有渗透**：层次间的过渡自然融合，无硬边界

**避坑指南**：
- ❌ 避免 3D 厚度、高光渐变
- ❌ 避免过于花哨的弹窗特效
- ✅ 坚持平面化设计 + 物理感动画

## 6. 主题与深度动效 (Theme & Deep Animation)

### 6.1 分层渗透主题切换

**切换逻辑**：墨水从编辑器中心点开始向外渗透
- **阶段 1（0-0.3s）**：背景色先变
- **阶段 2（0.3-0.6s）**：图标、按钮等 UI 元素变色
- **阶段 3（0.6-0.8s）**：正文文字最后过渡
- **曲线**：`cubic-bezier(0.22, 1, 0.36, 1)`，比标准曲线更丝滑
- **价值**：避免全屏一闪的突兀感，营造"墨水渗透"的仪式感

### 6.2 动态字体呼吸（实验性功能）

**效果**：AI 正在思考或生成内容时，界面标题字重（Weight）在 400 到 600 之间极其缓慢地波动
- **周期**：4-6 秒一个呼吸周期
- **幅度**：极其细微，不干扰阅读
- **开关**：用户可关闭
- **价值**：让界面感觉是"活的"，而非冷冰冰的静态模板
