## 1. 设计目标

本设计文档定义 easyStory 的核心创作流程功能，包括：

1. 创作起点的交互方式
2. 工作流节点系统的设计
3. 上下文注入机制
4. 审核与精修流程
5. 双工作模式（手动/自动）

**设计原则：**
- **平台思维**：不预设具体流程，用户自定义节点和流程
- **灵活性优先**：支持多层级配置，满足不同用户需求
- **渐进式复杂度**：新手用默认配置，高级用户深度定制

## 2. 创作起点：自由对话式设定

### 2.1 设计决策

**采用方案：自由文本对话**

用户通过自然语言对话确定创作方向，AI 理解并提取关键信息。

**理由：**
- 符合 AI 创作的自然交互方式
- 不限制创作自由度
- 避免结构化表单的僵硬感

### 2.2 交互流程

```
用户输入自由文本
  ↓
AI 理解并提取关键信息
  - 题材（玄幻/都市/悬疑...）
  - 文风（轻松/严肃/幽默...）
  - 主角设定
  - 配角设定
  - 世界观
  ↓
AI 追问细节（多轮对话）
  ↓
生成初始设定文档
  ↓
用户可以随时修改
```

### 2.3 实现要点

- 使用对话式 AI 接口
- 提取的信息存储为结构化数据（便于后续注入）
- 支持随时修改设定文档

## 3. 工作流节点系统

### 3.1 核心理念

**平台思维，用户自定义流程**

系统不预设固定流程（如"必须先大纲再章节"），而是提供节点构建能力，用户自由组合。

### 3.2 节点定义

每个节点代表一个创作步骤，可以关联：

| 组件 | 说明 | 是否必填 |
|-----|------|---------|
| **Skill** | 提示词模板 | 可选，没有则用临时 Prompt |
| **Hooks** | 自动化操作（如自动保存） | 可选 |
| **Agents** | 审核/精修助手 | 可选 |

### 3.3 节点配置示例

```yaml
- id: "outline"
  name: "生成大纲"
  type: "generate"
  skill: "skill.outline.xuanhuan"  # 可选
  
  hooks:
    after:
      - "hook.auto_save"
  
  auto_review: true
  reviewers:
    - "agent.consistency_checker"
  
  auto_proceed: false  # 需要人工确认
```

### 3.4 节点创建方式

**混合界面（第一阶段实现表单 + YAML）**

| 方式 | 适用用户 | 实现优先级 |
|-----|---------|-----------|
| 表单式配置界面 | 新手 | P0（第一阶段） |
| YAML 编辑模式 | 高级用户 | P0（第一阶段） |
| 可视化拖拽 | 所有用户 | P1（后续版本） |

**两种模式实时同步：**
- 表单修改 → 自动更新 YAML
- YAML 修改 → 自动更新表单

## 4. 上下文注入机制

### 4.1 设计目标

避免内容漂移和不连贯，通过注入相关上下文（大纲、前几章等）保持创作一致性。

### 4.2 三层优先级配置

**优先级：节点级 > 模式匹配 > 全局规则**

#### 层级 1：全局规则（Workflow 级别）

```yaml
workflow:
  context_injection:
    enabled: true
    default_inject:
      - type: "outline"      # 所有节点都注入大纲
      - type: "chapter_list" # 所有节点都注入章节目录
```

#### 层级 2：模式匹配规则

```yaml
context_injection:
  rules:
    - node_pattern: "chapter_*"  # 匹配所有章节节点
      inject:
        - type: "previous_chapters"
          count: 2  # 注入前2章
```

#### 层级 3：节点级覆盖

```yaml
- id: "chapter_10"
  context_injection:
    - type: "previous_chapters"
      count: 5  # 覆盖全局的2章
```

### 4.3 支持的注入类型

| 类型 | 说明 | 参数 |
|-----|------|------|
| `outline` | 大纲 | - |
| `chapter_list` | 章节目录 | - |
| `previous_chapters` | 前 N 章内容 | `count` |
| `character_profile` | 人物设定 | - |
| `world_setting` | 世界观设定 | - |
| `custom` | 自定义内容 | `source`, `query` |

### 4.4 实现复杂度

**难度：低**

配置解析和优先级合并的标准逻辑，实现直接。
