# 小说分析功能

| 字段 | 内容 |
|---|---|
| 文档类型 | 功能设计 |
| 优先级 | 🟡 MVP 建议调整 |
| 关联文档 | [创作设定](./06-creative-setup.md)、[上下文注入](./02-context-injection.md) |

---

## 1. 概述

用户上传已有小说，系统分析其文风、剧情、人物等特征，并自动生成模仿该文风的 Skill。包括：上传与章节切分、智能采样、分析报告、用户确认与试写、Skill 生成、与主流程的关系。

---

## 2. 核心挑战

- 一本小说可能几十万到上百万字
- LLM 上下文窗口有限（Claude 200K tokens ≈ 30-40 万字）
- 需要在分析质量和 token 消耗之间平衡

---

## 3. 分析流程（优化版）

```
用户上传小说
  ↓ 系统自动按章节切分
  ↓ 用户选择: 章节范围 / 智能采样
  ↓ 系统分析选中章节
  ↓ 生成分析报告
  ↓ 【用户确认/修正分析结果】
  ↓ 基于确认后的分析生成 Skill
  ↓ 【用户测试 Skill（试写一段）】
  ↓ 满意 → 保存 Skill / 不满意 → 调整重生成
```

### 3.1 智能采样规则

```yaml
采样策略:
  触发条件: 章节数 > 30
  采样规则:
    - 开头: 前 5 章（建立基调）
    - 中间: 均匀采样 10 章
    - 结尾: 后 5 章（高潮结局）
    - 总计: 约 20 章
```

---

## 4. 分析维度

| 维度 | 说明 | 示例 |
|-----|------|------|
| 文风特征 | 用词风格、句式特点、节奏感 | "用词华丽，多用成语；节奏明快" |
| 叙事视角 | 第一/第三人称，全知/限知 | "第三人称限知视角" |
| 情感基调 | 整体情感倾向 | "轻松幽默，偶有紧张" |
| 剧情结构 | 故事发展脉络 | "双线叙事" |
| 人物特征 | 性格和关系 | "主角：坚韧不拔" |

### 4.1 分析报告可编辑

```json
{
  "writing_style": {
    "vocabulary": "华丽，多用成语",
    "sentence_structure": "长短句结合",
    "rhythm": "节奏明快"
  },
  "user_notes": "这本书的特点是对话幽默，要保留"
}
```

## 5. 章节检测回退策略

```
1. 标准检测（优先）
   正则: "第[一二三四五六七八九十百千万零\d]+[章节回集卷]"

2. 通用模式检测（回退 1）
   正则: "^(Chapter|CHAPTER)\s+\d+"
         "^#{1,3}\s+"  (Markdown 标题)
         "^[-=]{3,}$"  (分隔线)

3. 固定长度切分（回退 2）
   每 3000-5000 字为一段（段落边界切分）
   UI 提示: "未检测到章节标记，已自动切分"

编码处理: UTF-8 → GBK/GB2312 → GB18030 → 报错
文件限制: 最小 1,000 字 / 最大 500 万字(50MB)
```

---

## 6. 来源确认与保留策略

- 用户上传时必须显式确认：对上传文本拥有使用权，或仅用于个人分析参考
- 默认长期保留**分析结果、采样摘要和生成的 Skill**，不长期保留整本原文
- 原始文件在分析完成后进入短期保留区；若用户未手动选择“保留源文件”，按保留策略自动清理
- Skill 试写阶段增加**长片段相似性检查**；若测试结果出现大段疑似复写原文，阻止直接保存，要求调整后重生

## 8. Skill 自动生成

### 8.1 质量评估

生成 Skill 后，系统自动用它生成测试内容：

```
系统: "已生成《金庸武侠》风格 Skill，测试内容："
  ↓ 【测试内容】
用户:
  - ✅ 满意，保存 Skill
  - ❌ 不满意，调整重新生成
  - ✏️ 手动修改 Skill prompt
```

### 8.2 生成的 Skill 示例

```yaml
skill:
  id: "skill.style.金庸风格"
  name: "金庸武侠风格"
  category: "chapter"
  prompt: |
    你是一位武侠小说作家，模仿金庸的写作风格。
    【文风特征】
    - 用词风格: {{ analysis.vocabulary }}
    - 句式特点: {{ analysis.sentence_structure }}
    请按照以上风格创作内容:
    {{ user_input }}
```

## 9. 与主流程的关系

### 9.1 双入口

```
1. 独立入口（全局）
   首页 → "小说分析" → 上传 → 分析 → 生成 Skill（全局可用）

2. 项目内入口
   项目 → "参考文风" → 上传 → 分析 → 生成 Skill（项目内可用）
   → 分析结果可作为上下文注入
```

### 9.2 Skill 作用域

```yaml
skill:
  id: "skill.style.金庸风格"
  scope: "global"              # global / project
  source: "analysis"           # analysis / manual / template
  source_novel: "射雕英雄传"
```

### 9.3 分析结果作为上下文注入

```yaml
context_injection:
  rules:
    - type: "style_reference"
      analysis_id: "xxx"
      inject_fields: ["writing_style", "narrative_perspective"]
```
